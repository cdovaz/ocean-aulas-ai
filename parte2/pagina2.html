Fundamentos de Prompt Engineering

Os blocos que compõem um prompt eficaz

🎯

Instrução Clara

Diga exatamente o que você quer que o modelo faça. Evite verbos vagos como "fale sobre". Use ações específicas: resuma, compare, classifique, gere, traduza.

📋

Contexto

Forneça informações de fundo necessárias: público-alvo, tom, formato, restrições e o objetivo da resposta. Quanto mais contexto relevante, melhor.

👤

Persona

Peça ao modelo para assumir um papel específico. Exemplo: "Atue como um mentor técnico sênior explicando para um desenvolvedor júnior."

📤

Formato de Saída

Especifique como quer a resposta: JSON, markdown, tabela, lista, e-mail, código. Isso reduz reformatação posterior.

📏

Restrições

Defina limites: tamanho máximo, palavras a evitar, nível de detalhe, o que não deve conter. Restrições aumentam a precisão.

✅

Exemplos

Mostre o padrão de entrada e saída desejado. Um ou dois exemplos costumam ser suficientes para alinhar o modelo.

💡

Prompt completo exemplo

Contexto + Persona + Instrução + Formato + Restrição: "Você é um analista de produto sênior. Resuma em 3 parágrafos curtos os principais feedbacks dos usuários abaixo. Use tom objetivo, formato markdown com bullets e não ultrapasse 200 palavras. [cole os feedbacks aqui]"
