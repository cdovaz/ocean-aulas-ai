⚠️ Regra de Ouro

Exemplos ruins ensinam o modelo a errar. Invista mais tempo na curadoria dos dados do que no treinamento em si.

📋 Formato Instruction Tuning

{ "instruction": "Resuma em 3 frases.", "input": "Texto longo...", "output": "Resumo aqui." }

💬 Formato Chat

{ "messages": [ {"role": "system", "content": "Você é um especialista jurídico."}, {"role": "user", "content": "Explique o artigo 5º."}, {"role": "assistant", "content": "Trata dos direitos fundamentais..."} ] }

✅ Boas Práticas

  • Quantidade: centenas a milhares de exemplos, conforme complexidade
  • Qualidade > quantidade: exemplos ruins ensinam o modelo a errar
  • Diversidade: cubra variações de phrasing, tamanho e edge cases
  • Consistência: mesmo formato, tom e profundidade
  • Balanceamento: evite que um tipo de exemplo domine
  • Dados reais: preferencialmente dados reais, não sintéticos genéricos
