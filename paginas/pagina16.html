Tipos de Arquiteturas de Agentes

Padrões de design para construir sistemas autônomos com LLMs

📋

Planner / Plan-and-Execute

Planeja todos os passos antes de executar. Ideal para tarefas complexas e de longo prazo.

Ex: planejar uma viagem completa antes de fazer reservas.
🔄

ReAct (Reason + Act)

Intercala raciocínio e ação em loop. O agente pensa, age, observa e repete.

Ex: buscar clima na API e sugerir guarda-chuva se chover.
🪞

Reflexão (Reflection)

Avalia suas próprias respostas, identifica erros e gera versões melhoradas.

Ex: escrever, revisar e corrigir um texto antes de entregar.
🛠️

Tool Use / Function Calling

Foca no uso de ferramentas externas. O LLM escolhe qual função chamar.

Ex: consultar CEP, calcular ou buscar em vector DB.
📚

RAG Agent

Recupera documentos antes de responder, reduzindo alucinações.

Ex: responder sobre um manual consultando trechos relevantes.
👥

Multi-Agent / Crew

Vários agentes especializados colaboram, cada um com um papel.

Ex: pesquisador, analista e redator criando um relatório.
🏗️

Hierárquica

Um supervisor coordena workers e integra os resultados.

Ex: orquestrador delega para agentes de vendas e suporte.
🕸️

Graph-based

Modela o agente como grafo de estados com ciclos e ramificações.

Ex: fluxo de suporte que escala, consulta FAQ ou abre chamado.
