Stack de Produção

Ferramentas que compõem o agente de relatórios

Banco de dados
🐘

Supabase

Armazena usuários, relatórios gerados, metadados e histórico de execuções. Também pode servir como fonte de dados estruturados.

Cache / Fila
⚡

Redis

Gerencia filas de tarefas assíncronas, cache de resultados intermediários e estado de execução do agente.

LLM
🧠

Gemini API

Responsável pela geração de texto, análise de dados e sumarização. Pode processar texto, tabelas e até imagens quando necessário.

Vector DB
🔍

Qdrant

Armazena embeddings de documentos internos. Permite recuperar contexto relevante para fundamentar os relatórios.

Orquestração
🕸️

LangGraph

Define o grafo de estados do agente: quais nós executam, em qual ordem, e como as decisões são tomadas.

🛠️ Por que essa stack?

Cada componente resolve um problema específico: persistência (Supabase), velocidade e filas (Redis), inteligência (Gemini), memória semântica (Qdrant) e fluxo de trabalho (LangGraph). Juntos, permitem construir um agente escalável, observável e fácil de manter.
